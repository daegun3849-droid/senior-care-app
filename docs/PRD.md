# AI Todo Manager - PRD (Product Requirements Document)

------------------------------------------------------------------------

# 1. 프로젝트 개요

## 1.1 배경

기존 할 일 관리 서비스는 단순 기록 중심이며, 사용자의 실제 생산성 향상에
대한 인사이트 제공이 부족함.\
본 프로젝트는 AI 기반 자동 구조화 및 요약 분석 기능을 통해 "기록 → 실행
→ 분석 → 개선"의 생산성 루프를 완성하는 것을 목표로 함.

## 1.2 목표

### 1) 제품 목표

-   자연어 기반 할 일 자동 구조화
-   직관적인 관리 환경 제공
-   데이터 기반 일/주간 생산성 분석 제공

### 2) 기술 목표

-   Next.js App Router 기반 확장 가능한 구조
-   Supabase 기반 인증 + DB + RLS 보안 설계
-   Gemini API 기반 안정적인 JSON 구조 응답 설계

### 3) KPI (MVP 기준)

-   가입 후 첫 할 일 생성까지 1분 이내
-   AI 생성 성공률 95% 이상
-   일일 요약 기능 사용률 50% 이상

------------------------------------------------------------------------

# 2. 사용자 정의

## 2.1 타겟 사용자

-   업무 및 개인 일정이 많은 직장인
-   학습 계획을 체계적으로 관리하고 싶은 사용자
-   생산성 개선에 관심 있는 사용자

## 2.2 사용자 시나리오

1.  회원가입 → 로그인
2.  자연어로 할 일 입력
3.  AI 자동 구조화 → 저장
4.  필터/정렬로 관리
5.  하루/주간 요약 확인

------------------------------------------------------------------------

# 3. 주요 기능 정의

## 3.1 인증 (Supabase Auth)

### 기능 범위

-   이메일/비밀번호 회원가입
-   로그인
-   로그아웃
-   세션 유지
-   비밀번호 재설정

### 보안

-   Row Level Security 활성화
-   user_id 기반 접근 통제
-   서버 컴포넌트에서 세션 검증

------------------------------------------------------------------------

## 3.2 할 일 관리 (CRUD)

### 데이터 필드

  필드           타입                      설명
  -------------- ------------------------- -----------
  id             uuid                      PK
  user_id        uuid                      FK
  title          text                      제목
  description    text                      설명
  created_date   timestamp                 생성일
  due_date       timestamp                 마감일
  priority       enum(high, medium, low)   우선순위
  category       text\[\]                  카테고리
  completed      boolean                   완료 여부

### 기능 요구사항

-   생성(Create)
-   목록 조회(Read)
-   단건 수정(Update)
-   삭제(Delete)
-   완료 토글
-   지연 상태 자동 계산 (due_date \< now AND completed = false)

------------------------------------------------------------------------

## 3.3 검색 / 필터 / 정렬

### 검색

-   title + description full text 검색
-   300ms debounce

### 필터

-   우선순위
-   카테고리
-   상태 (진행중 / 완료 / 지연)

### 정렬

-   우선순위순
-   마감일순
-   생성일순

------------------------------------------------------------------------

## 3.4 AI 할 일 생성

### 처리 흐름

1.  사용자 자연어 입력
2.  API Route 호출
3.  Gemini API 요청
4.  JSON schema 기반 응답 강제
5.  Zod 검증
6.  DB 저장

### 응답 스키마

{ "title": "string", "description": "string", "due_date": "ISO8601",
"priority": "high \| medium \| low", "category": \["string"\],
"completed": false }

### 예외 처리

-   JSON 파싱 실패 시 재요청
-   날짜 해석 실패 시 사용자 확인

------------------------------------------------------------------------

## 3.5 AI 요약 및 분석

### 일일 요약

-   오늘 완료 건수
-   남은 작업 수
-   주요 카테고리 요약
-   AI 텍스트 분석 결과

### 주간 분석

-   총 작업 수
-   완료율 (%)
-   카테고리별 분포
-   우선순위 분포
-   평균 완료 소요 시간

### 구현 방식

-   서버에서 기간 데이터 집계
-   통계 데이터 + raw 데이터 → Gemini 전달
-   카드 UI + 차트 시각화

------------------------------------------------------------------------
