# 내일 시작할 때 — 마무리 체크리스트

고생 많으셨습니다. 아래만 순서대로 확인하면 됩니다.

## 1. 앱 주소 (PC·모바일 공통)

- **프로덕션:** https://senior-care-app-pi.vercel.app  
- PC에서 `localhost` 안 쓸 때도 위 주소만 쓰면 됩니다.

## 2. 코드 / Git

- 프로젝트: `senior-care-app` 폴더
- 브랜치: `main` (원격과 맞춘 뒤 작업)
- 내일 열 때: `git pull origin main` 후 `npm install`(필요 시) → `npm run dev`

## 3. Supabase (데이터)

- **다중 체크·+1회** 쓰려면 `docs/migration-logged_at.sql` 전체를 Supabase **SQL Editor**에 붙여넣고 **Run** 한 번 실행했는지 확인.
- 아직이면 내일 먼저 실행 (에러 나면 해당 줄만 에러 보고 수정).

## 4. Vercel

- **배포:** GitHub에 push 되면 연동되어 있으면 자동 재배포. 수동일 때는 `npx vercel --prod`.
- **환경 변수:** 프로젝트 Settings → Environment Variables  
  `NEXT_PUBLIC_SUPABASE_URL`, `NEXT_PUBLIC_SUPABASE_ANON_KEY`, `GROQ_API_KEY`, `GOOGLE_GENERATIVE_AI_API_KEY`, 알림용 `VAPID_*`, `CRON_SECRET` 등이 Production에 들어있는지 확인.
- **Cron (무료 플랜):** `vercel.json` 기준 매일 한 번만 가능 → 현재 `/api/push-notify` **매일 20:00 UTC** 근처(한국은 새벽) 스케줄. 필요하면 내일 시간대 조정 검토.

## 5. PC 말하기(음성)

- Chrome 주소창 왼쪽 **자물쇠 → 마이크 → 허용**
- Windows 설정 → 개인 정보 → 마이크에서 Chrome 허용
- 문제 있으면 앱 알림 메시지(권한/마이크/네트워크 안내) 참고

## 6. 이어서 하고 싶은 일 (선택)

- 복약 알림 시간을 크론에 맞게 한국 시간으로 다시 조정
- 로컬 `localhost` 사용 시 포트 충돌 → `taskkill /f /im node.exe` 후 `npm run dev`
